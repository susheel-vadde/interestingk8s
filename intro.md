# Kubernetes Engineer Technical Assessment

Welcome to the hands-on technical assessment!

## Scenario
You've inherited a Java microservice that needs to be containerized and deployed to Kubernetes. The previous developer left behind broken configuration files that you need to fix.

## Your Mission
1. Fix the broken Dockerfile and build a working container
2. Deploy the application to Kubernetes
3. Troubleshoot and fix deployment issues
4. Configure the application properly with ConfigMaps and health checks
5. Scale and expose the service

## What You'll Need
- Docker and kubectl knowledge
- Kubernetes manifest debugging skills
- Java application understanding
- Problem-solving abilities

## Time Limit
60 minutes

## Evaluation Criteria
- Correct use of Docker commands
- Proper kubectl commands and troubleshooting
- Understanding of K8s concepts
- Systematic debugging approach

Click **Start** when you're ready!
